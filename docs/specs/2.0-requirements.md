# Pythoneer Wrangler 2.0 Requirements

## 1. Project Goal
To provide a low-latency, high-impact interactive dashboard that bridges FastAPI and Hardware Control (LEDs). The dashboard must handle concurrent audience input while maintaining high-priority admin controls.

## 2. Command Hierarchy
- **Admin (Speaker):** High-priority override via WebSocket headers. Can override any audience "Heckle" or "Color Vote."
- **User (Audience):** Commands are queued and subject to moderation.

## 3. The Queue System
- **Priority Queue:**
  - **Admin Text:** Immediate display.
  - **Audience Text:** Round-robin display with a 5-second cooldown per user.
- **Safety Moderator:**
  - Profanity filtering.
  - Duplicate detection.
  - Cooldown enforcement.

## 4. Network & Discovery
- **mDNS (Zeroconf):** Integration so attendees can access via `wrangler.local`.

## 5. Telemetry & Analytics
- **Frame Delta:** Real-time millisecond delay between "Command Sent" and "Hardware Acknowledged."
- **Power Draw Estimate:** Virtual wattage calculation based on LED brightness and color values.

## 6. Technical Constraints
- **Backend:** Python 3.11+ using FastAPI and uvicorn.
- **Hardware Interface:** `rpi_ws281x` or FastLED (via Serial/ESP32 bridge).
- **Latency Targets:**
  - UI Feedback: < 30ms.
  - Hardware Response: < 100ms.

## 7. Aesthetic
- **Cyber-Dark:** High-contrast, presentation-optimized (Tailwind CSS `slate-950` base).
